# 第 27 章自检题答案

1. **必须算被抢占的时间**。任务 i 的"响应时间" R_i = C_i + Σ(高优先级任务在 R_i 时间内的执行)。这是 RMS 严格分析的核心公式。Cortex-M 的 tail-chaining 等优化能让"被抢占的开销"接近 0，但不为 0。

2. (a) **任务集合不是最坏的**：周期不互相成倍数 / WCET 估算保守 / 实际 cache 命中率高 → 真实利用率比理论 WCET 总和低。(b) **大多数项目是软实时**，偶尔错过截止期没人会死，所以靠"95% 不超时"撑过去。RMS 的 69% 是**所有任务都按 WCET 跑且周期最坏组合**的极端理论值。

3. **可预测性 > 平均速度**。硬实时不能容忍 "99% 100 µs, 1% 5 ms" —— 那 1% 会让你撞车。关 cache 后平均变慢但 WCET 缩小到一个紧界。同样道理：Cortex-R 系列禁用乱序、要求双核锁步。

4. 中断响应延迟 < 5 µs (5% 周期)；抖动 < 1 µs (1% 周期，影响 PWM 占空比精度)；ISR 内执行时间 < 50 µs (留 50% 给应用任务)。建议：把 ADC + PI + PWM 全放在 ISR 里完成，避免任务级抖动。这种代码 Cortex-M4 + FPU + 168 MHz 是常见配置。

# 第 36 章自检题答案

1. (a) 状态 FF 单独一段，综合工具立刻识别 → 不会误推断 latch；(b) 转移逻辑单纯组合，工具知道是 MUX 树，能优化；(c) 输出逻辑独立，Moore vs Mealy 风格清晰，能选择是否插入流水段。新人写 1-process 容易把转移和输出混在时序块里 → 综合后变成"状态变完才输出"延迟一拍，行为偏离意图。

2. 综合工具看到 `if (cond) y = a;` 但 `else` 没赋值 → 推断 **Latch**：cond 为真时 y 跟着 a 变，否则保持。这通常不是你想要的，仿真和综合可能不一致，时序分析也变诡异。**默认值赋在 case 上面**或写完整 if-else 防御。

3. **Binary**：状态多但 FF 紧张时（ASIC、资源敏感场景）。**One-hot**：FPGA 上 FF 富裕、想要更快关键路径（每状态独立判断，无解码延迟）。**Gray**：跨时钟域的状态转移，相邻状态只差 1 位，单 bit 同步器即可。

4. 滤毛刺：在 START 状态我们只看到了"线变低"这一个边沿。中点（bit 时间的一半 = 7 个 baud16_tick）再确认一次，如果仍是低 → 真的是起始位；如果高 → 噪声，回到 IDLE。这是异步收发器的标配防误触发设计。

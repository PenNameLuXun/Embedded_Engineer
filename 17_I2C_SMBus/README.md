# 第 17 章　I²C / SMBus：两根线挂一群设备

> SPI 一对一全双工、有片选。I²C 走另一条极端：**只用两根线** + 地址寻址，能挂几十个设备。代价是慢、半双工、协议复杂。EEPROM、温度传感器、IO 扩展、各类配置寄存器 90% 走 I²C。
>
> **学完本章你应该能**：(1) 看懂 I²C 帧（起始 / 寻址 / 数据 / ACK / 停止），(2) 解释多主仲裁、时钟拉伸怎么工作，(3) 知道 I²C 为什么必须是开漏 + 上拉，(4) 区分 SMBus、PMBus、I3C 的差别。

---

## 17.1 物理层：两根开漏 + 上拉

```
                VCC
                 │
       Rp ╪    Rp ╪    （上拉电阻，典型 4.7 kΩ）
                 │
   ──────┬──────────  SDA  (双向数据)
                 │
   ──────┬──────────  SCL  (双向时钟)
         │           
       多个设备同时挂在这两根线上
       每个设备都是 open-drain，能拉低、能释放
```

**为什么必须开漏**：多个设备共享一根 SDA。如果某个推挽，它和别人就 "打架"了。开漏 → 任何一个拉低就低，全释放才高（**线与，wired-AND**），物理上完美支持总线共享。

**上拉值的取舍**：小 → 上升快但功耗大；大 → 省电但上升慢、容易被电容拉成斜坡。3.3 V 系统典型 2.2 kΩ ~ 10 kΩ。

## 17.2 一帧的解剖

```
       START   addr(7)         R/W   ACK    data(8)    ACK   ...   STOP
        ┌─┐   ┌──────────────┐ ┌─┐  ┌─┐    ┌──────┐   ┌─┐         ┌──┐
SDA  ───┘ └───┤ A6 A5...A0   ├─┤ │──┤ │────┤ D7..0├───┤ │─...─────┤  │
       SDA↓             ↓       ↓   ↓             ↓              SDA↑
SCL  ────╲___╱──╲_╱──╲_╱──╲_╱──╲_╱──╲_╱──╲_╱──╲_╱──╲_╱─────╲_╱──
       SCL still high   8 个时钟        9 个时钟         SCL high
```

**关键规则**：
- **数据只在 SCL 低期间改变**；SCL 高期间 SDA 改 = 信号（START / STOP）
- **START** = SCL 高时 SDA 下降沿
- **STOP**  = SCL 高时 SDA 上升沿
- **ACK**   = 接收方在第 9 个时钟把 SDA 拉低；NACK = 留高
- 数据 **MSB 先**（与 UART/SPI 默认 LSB 习惯相反，又一个混淆点）

地址默认 **7 位**（128 个地址，约 112 个可用，去掉保留），后跟 1 位 R/W：

| 字段                 | 含义                |
|----------------------|---------------------|
| START                | 总线占用开始        |
| Address[6:0]         | 目标设备 7 位地址   |
| R/W                  | 0=写，1=读           |
| (slave ACK)          | 从机响应            |
| Data byte 1          | 8 位                |
| (receiver ACK)       | 接收方响应          |
| ...                  | 多字节             |
| STOP / Repeated START | 结束 / 重新开始     |

### Repeated START

不释放总线就再发一个 START。用途：
- 写寄存器地址 → repeated start → 读那个寄存器的值（最经典：传感器配置 + 读）
- 多主时减少被抢的概率

### 10 位地址扩展

发 `1111_0XX0` (保留前缀 + 高 2 位 + W) → ACK → 第二字节 = 低 8 位 → ACK。能挂 1024 个地址但很少用。

## 17.3 速度档位

| 名称              | 速率       | 备注                       |
|-------------------|------------|----------------------------|
| Standard mode     | 100 kbit/s | 老设备默认                  |
| Fast mode (Fm)    | 400 kbit/s | 大多数现代设备              |
| Fast mode plus    | 1 Mbit/s   | 较新的传感器                |
| High-speed mode   | 3.4 Mbit/s | 实际很少见                  |
| Ultra Fast (UFm)  | 5 Mbit/s   | 单向                        |

速率上不去的根源：开漏 + 大电容 + 上拉电阻 → RC 时间常数限制上升沿。

## 17.4 多主仲裁

I²C 允许多个主同时存在。同时启动时，**冲突在 SDA 上自动解决**：

```
主 A 想发 1，主 B 想发 0。
A 释放（线高），B 拉低（线低）。
线上结果：低。
A 看到：我想发 1 但线是 0 → 我输了，主动退出。
B 看到：我发 0，线是 0 → 我赢，继续。
```

每位都比较一次，"最低地址 / 最低数据" 的主获胜。**所有主都开漏 + 同步采样** 是仲裁能工作的根本。

---

## 17.5 时钟拉伸 (Clock Stretching)

从机来不及处理时，可以**拉低 SCL 不让主释放**，给自己争取时间：

```
主 想让 SCL 高 ──→ 从 同时拉低 ──→ 线还是低
                                 主感觉到 SCL 没到高 → 等
                  从 处理完 ──→  释放 SCL ──→ 上升 → 继续
```

主必须**每次释放 SCL 后查实际 SCL 是否真的变高**，没变 = 从在拉伸 = 等。

这个机制让低速 / 单片机从机能跟上快主，但代价是主的状态机变复杂。某些主 IP 不支持时钟拉伸（早期 STM32 F1 著名 bug）。

## 17.6 I²C IP 核内部

```
┌────────────────────────────────────────────────────────────┐
│                                                              │
│   ┌──────┐   ┌──────────────────┐    ┌─────────┐             │
│   │ 总线 │   │  数据移位寄存器  │    │ SDA 驱动 │            │
│   │      │──→│                  │───→│ (开漏)  │── SDA      │
│   │      │   └──────────────────┘    └─────────┘             │
│   │      │                                                   │
│   │      │   ┌──────────────────┐    ┌─────────┐             │
│   │      │←──│ 地址匹配 / R-W   │    │ SCL 生成 │── SCL      │
│   │      │   └──────────────────┘    │ + 监听   │            │
│   │      │   ┌──────────────────┐    └─────────┘             │
│   │      │←──│ START/STOP 检测  │                            │
│   │      │   └──────────────────┘                            │
│   │      │   ┌──────────────────┐                            │
│   │      │←──│ FSM (主 / 从)    │                            │
│   │      │   └──────────────────┘                            │
│   └──────┘                                                   │
└────────────────────────────────────────────────────────────┘
```

**FSM 状态**（主侧粗粒度）：

```
IDLE → START_GEN → ADDR_TX → WAIT_ACK → 
   ┌→ DATA_TX → WAIT_ACK ─┐
   │                       │
   └─ DATA_RX → SEND_ACK ──┴→ STOP_OR_REPEATED → IDLE
```

加上仲裁失败 / 总线错 / 时钟拉伸超时几条转移线，就是工业级 I²C 主了。

## 17.7 SMBus / PMBus / I3C

| 名字   | 关系                              | 差别                              |
|--------|-----------------------------------|-----------------------------------|
| SMBus  | I²C 子集 + 命令格式 + 超时        | PC 主板上电池 / 温度走 SMBus       |
| PMBus  | SMBus + 电源管理协议层            | 数字电源、48 V 服务器系统          |
| I3C    | I²C 的"下一代"，向后兼容           | 推挽高速 + 中断回传 + 热插拔        |

I3C 是 MIPI 联盟在 2017 年推的，目前在传感器领域开始普及。

## 17.8 LM3S6965 I²C 实战速览

```c
#define I2C0_BASE       0x40020000u
#define I2C0_MSA        (*(volatile uint32_t *)(I2C0_BASE + 0x000u))
#define I2C0_MCS        (*(volatile uint32_t *)(I2C0_BASE + 0x004u))
#define I2C0_MDR        (*(volatile uint32_t *)(I2C0_BASE + 0x008u))
#define I2C0_MTPR       (*(volatile uint32_t *)(I2C0_BASE + 0x00Cu))

void i2c0_write_byte(uint8_t addr7, uint8_t reg, uint8_t data)
{
    I2C0_MSA = (addr7 << 1) | 0;        /* 写 */
    I2C0_MDR = reg;
    I2C0_MCS = 0x3;                     /* START + RUN */
    while (I2C0_MCS & 1u) {}            /* BUSY */
    I2C0_MDR = data;
    I2C0_MCS = 0x5;                     /* STOP + RUN */
    while (I2C0_MCS & 1u) {}
}
```

完整代码 `code/i2c_demo.c`。**QEMU 对 LM3S I²C 仿真很弱**，代码主要作为模板。在真硬件上你可以用 24LC256 / EEPROM、PCF8574 IO 扩展、SHT3x 温湿度等练手。

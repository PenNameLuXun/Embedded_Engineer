# 第 45 章自检题答案

1. **"用了高级抽象的代码运行时性能 ≈ 手写底层代码"**。例：迭代器 `arr.iter().filter(|&&x| x > 0).sum::<i32>()` 编译后与手写 for 循环生成的汇编**几乎一致**。Trait dispatch 静态化、泛型单态化、内联让运行时无虚函数表查询、无类型检查、无垃圾回收。

2. **`heapless`** crate 提供 `Vec<T, N>`、`String<N>`、`Deque<T, N>` —— 容量在编译期确定，栈上分配。或者自定义 allocator 配 `alloc` crate。**90% 嵌入式 Rust 项目用 heapless**：没堆碎片、可分析、WCET 可估。

3. **RTIC 是静态优先级 + 编译期资源分析**：所有任务、共享资源、优先级在编译时确定，编译器证明无死锁、不会优先级反转 (用 Stack Resource Policy)。**FreeRTOS 是运行时动态**：任务在运行时创建 / 删除，需要 mutex 等开销，死锁靠开发者避免。RTIC 更安全但灵活性低。

4. Embassy 的 `async fn` 被编译成**状态机**（一个枚举 + match）。每次 `.await` 是状态机的一个分支：保存当前位置 → 让出 CPU → 事件来时跳回。**所有任务共享一个栈**（执行器调度），不需要为每个任务预分配栈空间。RAM 占用比传统 RTOS 任务低 5-10×。

# 第 05 章　数字电路与时序

> 上一章是模拟，本章回到数字。第 01 章已经讲了组合 vs 时序逻辑，这章深入"时序" —— **setup/hold 时间、亚稳态、时钟域**，以及为什么时钟域穿越 (CDC) 是数字设计最容易出 bug 的地方。
>
> **学完本章你应该能**：(1) 解释建立/保持时间和亚稳态的关系，(2) 知道什么是同步器、为什么单级 FF 不够，(3) 知道异步 FIFO 大概怎么实现，(4) 看出代码里的潜在毛刺 (glitch) 与 hazard。

---

## 5.1 D 触发器再看一次：完整时序参数

第 01 章我们说"D 触发器在时钟边沿采样 D，锁到 Q"。完整的参数：

```
                    ┌──── T_setup ──┬─ T_hold ─┐
   D ───────────────█████████████████████████──────  
                                    │
                                    ↑ 时钟上升沿
   clk ___________________________┌──┐___________
                                  │  │
                                  ▼   
                          T_clk_to_Q 后
   Q ─────────────────────────────────█████████──
```

| 参数             | 含义                                    |
|------------------|-----------------------------------------|
| `T_setup` 建立时间 | 时钟边沿**之前**，D 必须稳定多久        |
| `T_hold`  保持时间 | 时钟边沿**之后**，D 还要保持多久        |
| `T_clk_to_Q`      | 边沿后多久 Q 才呈现新值                |
| `T_pd` 组合传播延迟 | 组合逻辑路径上的传播时间               |

### 时钟频率上限

两个 FF 之间夹一个组合逻辑路径：

```
  FF_A.Q ─→ [comb logic] ─→ FF_B.D
       └─ T_clk_to_Q ─┘ └─ T_pd ─┘ └─ T_setup_B ─┘
```

要 FF_B 能在下一个时钟边沿正确采到，必须：

```
T_clk_to_Q + T_pd + T_setup <= T_clk_period
```

→ **最大频率 = 1 / (T_clk_to_Q + T_pd_max + T_setup)**

这就是 FPGA 和芯片厂讲的 "Fmax / 时序收敛"。综合工具会算每条路径，找出最慢的 → "关键路径 (critical path)"。

---

## 5.2 违反时序：亚稳态

如果 D 在边沿前 / 后没稳够时间（setup / hold 违反），FF 内部锁存器电压可能落在禁区 (VIL ~ VIH 中间)，需要额外随机时间才能稳定到 0 或 1，叫 **亚稳态 (metastability)**。

```
       ↑边沿
   D ──╲╱╳╳╳╳──   ← D 在边沿附近变了
                
   Q ──────?????????─?─?──   ← Q 卡在禁区，
                              一段不定时间后随机停 0 或 1
```

后果：
- 下游电路把 Q 当 0 用，另一路把 Q 当 1 用 → 状态机分裂
- 通过组合逻辑链可能传播 → 多个 FF 一起亚稳

亚稳态在同一时钟域内通常被时序约束自动避免（综合工具不会让你的关键路径违反）。**真正的麻烦在跨时钟域**。

---

## 5.3 跨时钟域 (Clock Domain Crossing, CDC)

两个独立时钟，不能保证它们的边沿对齐：

```
  clk_a  ╱──╲___╱──╲___╱──╲___
  clk_b  ──╱──╲___╱──╲___╱──╲_
```

如果 A 域产生的信号直接接到 B 域的 FF，**B 域 FF 一定会随机违反 setup/hold** → 亚稳态频繁发生。

### 同步器：两级 FF (Two-Flop Synchronizer)

最简单的解法：在 B 域用两级（甚至三级）串联 FF 接收 A 域信号。

```
  A_signal ──┬─→ FF1 ─→ FF2 ─→ B_用户逻辑
             │   ↑       ↑
            (无)│       │
                 clk_b   clk_b
```

原理：FF1 可能亚稳，但**有一整个时钟周期**给它稳。到 FF2 输入时已大概率稳到 0 或 1。两级把"还在亚稳"的概率降到工程可接受（**MTBF** 计算）。

**这是 CDC 的最基础姿势**，但**只对单 bit 控制信号有效**。多 bit 数据用同步器是错的（位间偏斜会出错）。

### 多 bit 跨时钟域 → 异步 FIFO 或握手

- **异步 FIFO**：用 Gray code 指针 + 两级同步器。Gray code 一次只变 1 位，单 bit 同步即可。
- **握手 (Handshake)**：发送侧 `req`，接收侧 `ack`，两个信号各自同步。慢但简单。
- **Pulse synchronizer**：脉冲穿越，用 toggle + 同步 + 边沿检测。

第 38 章片上总线（AXI 异步桥）会用到。

## 5.4 复位：同步 vs 异步

复位信号也分两种风格：

| 类型     | 行为                                  | 优点           | 缺点                          |
|----------|---------------------------------------|----------------|-------------------------------|
| 异步复位 | 撤销复位不需要等时钟                  | 简单，无需时钟也能复位 | 撤销时若靠近时钟边沿会亚稳 |
| 同步复位 | 复位和撤销都在时钟边沿                 | 时序干净       | 复位时必须有时钟           |

**实战推荐**：**异步置位 + 同步释放 (asynchronous assertion, synchronous deassertion)**。撤销复位时经过同步器，避免亚稳进入主逻辑。Cortex-M 内部复位电路就是这种方式。

---

## 5.5 毛刺 (Glitch) 与冒险 (Hazard)

**组合逻辑**多条路径汇合时，传播延迟不同 → 输出会有短暂错误尖峰。

例：选 `Y = A·B + Ā·C`，当 A 从 1 变 0、B=C=1 时，Y 理论上一直 1，但途中可能短暂 0：

```
A   ───┐___
B   ─────────
C   ─────────
       
Y   ─────╲_/───   ← 一个毛刺
```

**关键点**：
- 毛刺只对 **组合逻辑输出直接接到外部 / 用于异步触发** 时才是问题。
- 进入 FF 的 D 上有毛刺，**没事** —— 时钟边沿采样时它已经稳了。
- 所以"在 FF 边界处取信号" / "用同步逻辑" 几乎天生免疫毛刺。

这是为什么数字设计的金科玉律是：**所有的信号都从 FF 出，到 FF 入**。

## 5.6 FSM 编码：Mealy vs Moore

第 01 章引入了 FSM。实战写时分两风格：

### Moore 机
输出只看**当前状态**：
```
                  state
   inputs ─→ [next_state logic] ──→ FF ──┬─→ [output logic] ─→ outputs
              ↑___________________________│
```

### Mealy 机
输出看**当前状态 + 输入**：
```
   inputs ─┬─→ [next_state logic] ──→ FF ─┐
           │                              │
           └──→ [output logic] ←──────────┤ ─→ outputs
```

| 维度         | Moore                 | Mealy                  |
|--------------|-----------------------|------------------------|
| 输出毛刺     | 没有（FF 之后）       | 有（输入直接进组合）    |
| 状态数       | 多                    | 少                      |
| 响应延迟     | 一拍                  | 当拍                    |

工业上**优先 Moore**，写得"笨"但稳。第 36 章会写 Verilog 实战。

## 5.7 流水线在 RTL 层是什么样子

第 02 章流水线讲的是 CPU。在 RTL 设计里，流水线就是 **把一长串组合逻辑切几段，中间各塞一组 FF**：

```
切之前： in ─[comb logic 长]──→ out      Fmax 低（关键路径长）

切之后： in ─[seg1]─FF─[seg2]─FF─[seg3]─FF─→ out
                    pipe     pipe     pipe
        Fmax 高（每段短），但延迟 (latency) 多了几拍
```

吞吐率 ↑，但 latency ↑。第 37 章 AXI 协议里你会看到大量这种流水设计。

## 5.8 实践：写 Verilog 时该养成的习惯

提前剧透第 35、36 章会展开，但这里先种下种子：

1. **永远用同步复位（或异步置位 + 同步释放）**，不要写裸异步复位的逻辑。
2. **时序逻辑用 `always_ff @(posedge clk)`，组合逻辑用 `always_comb`**（SystemVerilog）；纯 Verilog 用 `always @(posedge clk)` + `always @(*)`。
3. **时序逻辑里只用非阻塞赋值 `<=`，组合逻辑里只用阻塞赋值 `=`**。混用是经典坑。
4. **跨时钟域信号必须显式标注**，过同步器；多 bit 用 FIFO 或握手。
5. **状态机用枚举 / 参数定义**，不要硬编码状态值。

---

## 5.9 本章小结

- D 触发器的 setup/hold 决定 Fmax。综合工具帮你算关键路径。
- 违反 setup/hold → 亚稳态，本时钟域内综合工具能避免；跨时钟域必须做 CDC。
- 单 bit CDC → 两级 FF 同步器。多 bit CDC → 异步 FIFO 或握手。
- 异步复位要 "异步置位 + 同步释放"。
- 毛刺只在组合输出直接用时才是问题，FF 后天生干净。
- FSM 优先 Moore；流水线 = 切组合 + 塞 FF。

# 第 39 章自检题答案

1. 工具做了：(a) 常量传播 + 死代码删除；(b) 资源共享（多个加法器变成一个共用）；(c) 寄存器重定时（把 FF 沿关键路径搬动）；(d) FSM 重编码（binary → one-hot 或反过来）；(e) 把局部信号优化掉。**功能上等价**，但网表和你的代码 1:1 已经看不出来。这是 STA / 调试时要看综合报告而不是源码的原因。

2. 不一定。其它常见原因：(a) 关键路径太长（组合逻辑链太深）→ 流水线打段；(b) routing congestion → 信号绕远路；(c) 跨 die / cross-clock-region → 走 SerDes；(d) FPGA 部分高负载，工具给不到足够路由资源。**单纯降频是兜底**，更好是从设计上分段。

3. (a) 异步复位释放路径；(b) CDC 同步器之间（让工具不把 setup/hold 当违反算）；(c) 调试信号（如 ILA 引出脚）；(d) 静态配置信号（开机一次写死后不变）。**不要随手加** —— 一个错误的 false_path 能让真正违反的路径漏掉。

4. (a) 不打扰被测系统：printf 走串口、堆栈、外设，可能改变时序；(b) 多信号同时采样：能看 32+ 信号同步关系；(c) 触发条件复杂：组合触发"valid=1 且 ID=5"才采；(d) 高速：百 MHz 采样实时；(e) 不占外设：UART/JTAG 可能正在调试别的东西。**FPGA 内嵌 ILA = 板内逻辑分析仪**。

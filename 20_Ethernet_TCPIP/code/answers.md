# 第 20 章自检题答案

1. ARP cache 满：旧条目被 LRU 替换，临时多发几次广播，性能轻微下降。**ARP 投毒**：攻击者发假 ARP Reply，让你以为 gateway 的 MAC 是攻击者 → 流量被劫持。防御：静态 ARP / 网络隔离 / 802.1X 认证。

2. 不能直接建立。Server 在 SYN_RCVD 状态下等待 ACK，超时（默认 ~1 分钟，重传 5 次）后回 RST 关闭半开连接。Client 也会因没收到任何 ACK 重传第三个 ACK（但 client 状态是 ESTABLISHED，会立即开始发数据，server 看到非 SYN 报文回 RST）。

3. 三类：(a) 应用层有自己的可靠性 / 重传机制（QUIC、TFTP、应用层 ACK）；(b) 实时性比完整性重要（VoIP、视频通话，丢帧好过卡顿）；(c) 简单请求-响应（DNS：1 次重试足够，下游用户重试）。

4. 1980 年代以太网技术约束：CRC32 误码率与帧长有关，1500 字节是检错能力与吞吐的平衡。Jumbo frame (9000B) 在内部数据中心常用 → 减少包头开销提高吞吐，但**不是标准**，跨网段不兼容。所以广域网保持 1500 兼容。

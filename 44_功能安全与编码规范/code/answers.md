# 第 44 章自检题答案

1. ASIL D 要求**残余风险 < 10⁻⁸ / 小时**（每 1 亿小时一次故障）。汽车要求每年开 ~500 小时 × 寿命 15 年 × 1 亿台 = 50 亿台·年 → 仍只能容忍 ~50 个失败实例 / 全球。这就是为什么不能靠"跑跑测试"通过，必须形式化和冗余。

2. (a) 栈深度不可预测 → WCET 分析失败；(b) 栈溢出后果严重；(c) 大多数递归算法都能改成迭代。MISRA 选择"完全禁止"是保守 + 清晰边界。航天 DO-178C 同样禁。

3. 现实是**比对到不一致就触发 fault**，没"瞬态容忍"。两个核 deterministic 跑同一份代码、同一份输入，输出必须 bit-perfect 相同。如果有 jitter（如 PLL 不稳）那是硬件设计问题，要解决而不是宽容。瞬态故障由 ECC、奇偶校验等更底层机制兜底。

4. (a) 形式化证明的是**模型**正确，模型可能不准确反映硬件 (cache / 时序 / 电源)；(b) 工具 + 编译器有 bug，形式化的源码 + 错的编译器 = 错的二进制；(c) 形式化代价大，多数项目只对**关键内核**做，外围测试覆盖。最佳实践：**两者结合**。

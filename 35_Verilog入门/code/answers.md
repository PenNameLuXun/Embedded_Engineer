# 第 35 章自检题答案

1. 改为 `cnt <= cnt + 1;`。原因：时序逻辑里阻塞 `=` 会让"读"和"写"按代码行顺序执行，多 always 块之间产生竞争，仿真结果不稳定，综合常常和仿真不一致。非阻塞 `<=` 表达"在时钟边沿那一刻全部一起更新"的 D 触发器物理语义。

2. 不在任何 always 里赋值 = 没人驱动它 = **悬空**。综合工具会优化掉或报警。仿真里它停留在 `x` (未知)。

3. **没有直接对应**。`#10` 是仿真延迟，告诉模拟器"过 10 个 timescale 单位再做下一步"。硬件里时间由时钟边沿驱动，没有"延迟 N 纳秒"的指令。`#10` 在综合时被忽略或报错。

4. 功能等价（都是组合逻辑赋值）。但 `assign` 只能用一行表达式，`always_comb` 可以有 if / case / 多语句。新代码偏好 `assign` 用于简单连接，`always_comb` 用于复杂组合逻辑（如多路选择器、解码）。

# 第 24 章自检题答案

1. **二值信号量**：0/1 计数，常用于 ISR→Task 事件通知，没有所有权概念。  
   **计数信号量**：N 阶计数，用于资源池。  
   **互斥量**：1 阶但带**所有权 + 优先级继承**。只有拿者能释放（API 检查），且持有时被高优先级任务等时自动提升优先级。

2. SysTick 中断本身优先级可以很高（被早期 ISR 用 NVIC 配置过）。在 SysTick 里直接做上下文切换 = 在一个可能被高优先级 ISR 嵌套的位置切栈，极不安全。**PendSV 优先级配成最低**，保证它一定在所有 ISR 完成后才执行 → 切换发生在系统"安静"的时刻。这是 ARM 设计 PendSV 的本意。

3. 不是。优先级反转不需要循环依赖，只需要"高优先级 task 被低优先级 task 间接挡住"。Deadlock 需要循环资源依赖。但二者都可能让系统不响应，调试时容易混淆。

4. **CPU 跑别的 Ready task**。`delay_ms` 实际是把当前 task 切到 Blocked 状态 + 在调度器里记录 "delta tick = 100"。每个 tick 检查 100 → 99 → ... → 0 时唤醒。期间 CPU 跑 ready task；都不 ready 就跑 idle task（通常 WFI 睡眠）。这与裸机 `delay_ms` 死等完全相反，是 RTOS 价值的核心。
